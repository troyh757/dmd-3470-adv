Responsive Images & Backgrounds

Modern web design requires serving optimized images across a vast range of devices and screen sizes. This exercise explores three complementary techniques for responsive imagery: the <picture> element for art-directed images, CSS background-size for controlling background image presentation, and image-set() for resolution-aware background images.

1 The <picture> Element

Learning Objectives

  • Understand when and why to use the <picture> element for art direction
  • Implement media queries within <source> elements for responsive image switching
  • Provide appropriate fallback images for older browsers

Task

The <picture> element provides explicit control over which image is displayed at different viewport sizes. Using the three images in the images/ folder (art-medium.jpg, art-large.jpg, art-extra-large.jpg), implement a responsive picture element in the placeholder to the right:

  • Displays the large image when the viewport is at least 945px wide
  • Displays the extra large image when the viewport is at least 1100px wide
  • Provides a default/fallback image for smaller viewports or older browsers

Hint: Research the media attribute on <source> elements and the proper order of source declarations.

2 Background-Size (Contain vs. Cover)

Learning Objectives

  • Understand the difference between background-size: contain and background-size: cover
  • Apply appropriate background sizing strategies for different use cases
  • Implement responsive background images using media queries

Task

Create two visual demonstrations showing the difference between contain and cover. The containers (.bg-contain-demo and .bg-cover-demo) are provided below.

Your implementation should:

  • Apply different background images from the images/ folder (bg-small.jpg, bg-medium.jpg, bg-large.jpg) at different viewport breakpoints
  • Use background-size: contain on the first demo to show the entire image (may leave empty space)
  • Use background-size: cover on the second demo to fill the container (may crop the image)
  • Include background-position: center and background-repeat: no-repeat for both

Contain: Full image visible, may leave empty space

Cover: Fills container, may crop image edges

Hint: Consider which sizing approach is better for logos vs. hero images.

3 Image-Set for High-DPI Displays

Learning Objectives

  • Understand device pixel ratio and high-DPI (Retina) displays
  • Implement image-set() to serve appropriate resolution images
  • Optimize image delivery for different screen densities

Task

The image-set() function allows browsers to select the most appropriate image based on the device's pixel density. Modern smartphones and laptops often have 2x or 3x pixel densities, requiring higher resolution images to appear crisp.

Implement image-set() on the .image-set-demo container using the pattern images:

  • pattern-1x.jpg for standard displays (1x)
  • pattern-2x.jpg for high-DPI displays (2x)
  • pattern-3x.jpg for ultra-high-DPI displays (3x)

This background uses image-set() for resolution switching

Hint: Research the modern CSS image-set() syntax. You do not need vendor prefixes for this exercise.

Note: To test this properly, you'll need a high-DPI display or use browser DevTools to simulate different pixel ratios.
